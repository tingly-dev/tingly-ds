# Wails 3 Shell — Understanding Report

The Go implementation was originally isolated below `wails3-deepseek/` while a
separate agent explored Tauri. It has since been promoted to the repository
root and imported into branch `main` as focused source and test commits. The
root contains only the Go/Wails implementation, its build metadata, tests,
product documentation, and SDLC records.

The application has no existing UI or business logic to integrate with. Its
smallest useful architecture is a single Go process and one system WebView.
The remote DeepSeek site remains responsible for login state and chat
behaviour; Wails provides desktop lifecycle and menu-bar affordances only.

The high-risk integration points are external navigation, popup/login flows,
downloads, and accidentally exposing native methods to untrusted remote page
JavaScript. The implementation should therefore register no Go services,
disable development tools in release use, and keep all native actions behind
tray callbacks rather than page-callable bindings.
