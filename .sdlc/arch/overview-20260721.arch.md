## Overview

The repository root is a small Go/Wails 3 desktop shell that loads DeepSeek
Chat in the operating-system WebView and is primarily controlled from the
system tray. The earlier Tauri documents are retained only as historical
alternative research; there is no Rust implementation in this tree.

## Components

| Component | Location | Purpose |
| --- | --- | --- |
| Native shell | `main.go` | Application, window lifecycle, tray actions, and quit flow |
| Navigation policy | `navigation.go` | Validates bridge origins and external URLs |
| Embedded assets | `assets.go`, `build/icons/` | DeepSeek tray and application artwork |
| Verification | `*_test.go` | URL-policy and embedded-image tests |
| Build system | `Taskfile.yml`, `build/` | Development, release build, icon, and package tasks |

## Dependencies

- Go 1.25+ — compiles the application and tests.
- Wails 3 `v3.0.0-alpha2.114` — native window, WebView, tray, and packaging.
- macOS WebKit — renders the remote DeepSeek application.

## Data Flow

The Go process creates a persistent WebView pointing at
`https://chat.deepseek.com/`. DeepSeek owns page rendering, authentication,
storage, and network traffic. Native code owns window lifecycle and tray
actions. A narrowly validated raw message may open an unrelated HTTP(S) URL in
the operating-system browser.

## Key Patterns

- Remote-first WebView: no duplicated chat UI.
- Least privilege: the remote page receives no registered Go service.
- Hide-on-close: closing preserves the session and process for fast reopening.
- Domain navigation policy: DeepSeek anchor navigation stays embedded;
  unrelated destinations open in the default browser.

## Integration Points

- Wails application, window, tray, and raw-message APIs.
- macOS WKWebView.
- DeepSeek web application over HTTPS.
- Operating-system tray and default browser.
- Lobe Icons source artwork and the macOS icon toolchain.
