## Overview

The repository root is the Go module for a macOS-first Wails 3 desktop shell
around DeepSeek Chat. The implementation was moved up from its temporary
`wails3-deepseek/` development directory so build commands now run directly
from the repository root.

## Components

| Component | Location | Purpose |
| --- | --- | --- |
| Native entry point | `main.go` | App, window, tray, and lifecycle setup |
| Native artwork | `assets.go`, `build/icons/` | Embedded tray icon and generated macOS iconset |
| Navigation policy | `navigation.go` | Decide which URLs stay embedded |
| Build metadata | `Taskfile.yml`, `build/` | Wails development and packaging |
| Product documentation | `README.md` | Setup, use, limitations, and security model |

## Dependencies

- Go toolchain — compiles the application and its tests.
- Wails 3 — native window, WebView, system tray, and packaging.
- Operating-system WebView — WebKit on macOS and WebView2 on Windows.

## Data Flow

The Go process creates one persistent WebView whose initial URL is
`https://chat.deepseek.com/`. DeepSeek owns authentication, rendering, and
network traffic. Native code owns only the window lifecycle, system tray,
reload, and quit actions.

## Key Patterns

- Remote-first UI: do not clone or inject a chat frontend.
- Least privilege: register no callable Go services for remote content.
- Persistent window: closing hides the window so its WebView session survives.
- Root module: source, tests, build metadata, and documentation share one
  repository root.

## Integration Points

- Wails 3 application and tray APIs.
- macOS WKWebView and menu-bar activation policy.
- DeepSeek Chat over HTTPS.
- Operating-system default browser for unrelated external destinations.
- Lobe Icons source assets for the DeepSeek application and tray marks.
